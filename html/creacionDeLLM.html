Documentación Técnica: Creación de ChefBot (Creación de un LLM)

Este manual documenta paso a paso el proceso de creación, entrenamiento y despliegue de ChefBot, un modelo de lenguaje ajustado (Fine-Tuned) para actuar como experto culinario, utilizando tecnología LoRA y cuantización de 4 bits.

1. Configuración del Entorno de Trabajo

El primer paso consistió en preparar el directorio de trabajo y aislar las dependencias del proyecto.

Creación del Entorno Virtual

Navegamos al directorio del proyecto y creamos un entorno virtual de Python para mantener limpias las instalaciones.

Navegación al directorio del proyecto.

Comando para crear el entorno virtual (venv).

Activación del Entorno

Una vez creado, se activó el entorno virtual. Esto se verifica al ver (venv) al inicio de la línea de comandos.

Instalación de PyTorch y CUDA

Para aprovechar la GPU NVIDIA RTX 5060, se instaló la versión "nightly" de PyTorch compatible con CUDA 13.x.

Verificación del Stack Tecnológico

Antes de proceder, validamos que Python pudiera acceder a la GPU. Mediante un script de comprobación, confirmamos que la versión de Torch (2.12.0.dev) detecta correctamente CUDA y reconoce la tarjeta NVIDIA GeForce RTX 5060 Laptop GPU con arquitecturas soportadas hasta sm_120.

Validación exitosa de la disponibilidad de CUDA y detección del hardware.

2. Generación y Validación del Dataset

El modelo necesita datos de calidad para aprender su nuevo rol.

Script de Generación

Se ejecutó el script generar_dataset.py para crear un conjunto de datos sintético.

Estructura del Dataset

El archivo generado recetas_chefbot_500.jsonl contiene pares de "instrucción" y "output". Aquí podemos ver cómo el modelo aprenderá a responder peticiones específicas de cocina, como recetas keto o menús completos.

Vista previa de los datos de entrenamiento (formato JSONL).

3. Entrenamiento del Modelo

Esta es la fase crítica donde el modelo base aprende los nuevos conocimientos.

Configuración del Entrenamiento

En el script train_chefbot.py, se configura el modelo base Microsoft/Phi-3-mini-4k-instruct. Se utiliza QLoRA (configuración de 4 bits con BitsAndBytesConfig) para reducir el consumo de memoria y permitir el entrenamiento en una GPU de laptop.

Configuración de carga en 4 bits y selección del modelo base.

Ejecución del Entrenamiento

Al lanzar el script, se observa cómo cargan los pesos y comienza el descenso de la pérdida (loss), indicando que el modelo está aprendiendo.

Monitorización de Hardware

Durante el proceso, el Administrador de Tareas confirma que la NVIDIA GeForce RTX 5060 está siendo utilizada al 100% de su capacidad de cómputo, con un uso de VRAM cercano a los 3.8 GB, lo cual confirma la eficiencia de la cuantización de 4 bits.

Finalización

El entrenamiento concluye exitosamente tras completar las 500 iteraciones (epochs configuradas).

Artefactos Resultantes

Como resultado, se genera la carpeta chefbot-lora, que contiene los adaptadores (pesos entrenados) y los checkpoints del proceso.

4. Fusión del Modelo (Merge)

Para facilitar el uso, fusionamos los adaptadores LoRA con el modelo base.

Proceso de Fusión

El script de fusión carga el modelo base y le aplica los cambios aprendidos permanentemente.

Modelo Final

En la carpeta chefbot-merged obtenemos el modelo final listo para producción, incluyendo el archivo model.safetensors de ~7.4 GB (modelo completo).

5. Pruebas de Inferencia

Finalmente, interactuamos con el ChefBot terminado para verificar tanto su conocimiento como sus restricciones.

Código de Chat

El sistema incluye instrucciones estrictas (System Prompt) para mantener al bot en personaje.

Validación de Respuestas de Dominio (Happy Path)

Se realizó una prueba funcional para verificar que el modelo adquirió conocimientos culinarios. Al preguntar "¿qué puedo hacer con patatas?" y pedir consejos sobre asado, ChefBot respondió con sugerencias coherentes, técnicas precisas y el tono servicial definido en el entrenamiento.

Interacción exitosa: ChefBot ofrece recetas y técnicas de cocina.

Prueba de Seguridad (Guardrails)

Se probó la robustez del modelo intentando hacer preguntas técnicas fuera de contexto (sobre puertos DNS). El modelo identificó correctamente que eso no corresponde a su rol y rechazó la respuesta amablemente, reafirmando su identidad como ChefBot.

Interacción de seguridad: El modelo se niega a responder temas no culinarios.
